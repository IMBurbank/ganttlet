# Agent Summary — Issue #9: Input Validation and Error Feedback

## What Changed and Why

### New Files
- **`src/utils/taskFieldValidation.ts`** — Pure, framework-agnostic validation functions:
  - `validateTaskName(value)` — rejects empty/whitespace-only names
  - `validateDuration(value)` — rejects non-numeric input and values < 1
  - `validateEndDateAfterStart(startDate, endDate)` — rejects end before start
  - `validateStartDateBeforeEnd(startDate, endDate)` — rejects start after end

- **`src/utils/__tests__/taskFieldValidation.test.ts`** — 20 unit tests covering all validators, including the 6 required cases: empty name, whitespace name, zero duration, negative duration, non-numeric duration, and end-before-start date.

### Modified Files
- **`src/components/table/InlineEdit.tsx`** — Added:
  - Optional `validate?: (value: string) => string | null` prop
  - `useId()` for a stable error element ID (React 19)
  - `error` state to track validation failure
  - `aria-invalid` and `aria-describedby` on the `<input>` when invalid
  - Inline error `<span role="alert">` displayed below the input when invalid
  - Whitespace trimming for `type="text"` fields before validation and save
  - On validation failure: stays in editing mode (does not call onSave)
  - On Escape: clears error and exits editing mode
  - Clears error on every keystroke to avoid stale messages

- **`src/components/table/TaskRow.tsx`** — Added:
  - Import of the four validation functions
  - `validate={validateTaskName}` on the name field InlineEdit
  - `validate={validateDuration}` on the duration field InlineEdit
  - `validate={v => validateStartDateBeforeEnd(v, task.endDate)}` on startDate
  - `validate={v => validateEndDateAfterStart(task.startDate, v)}` on endDate

- **`src/index.css`** — Added:
  - `.inline-edit-input.inline-edit-error` — red border + red glow for invalid input
  - `.inline-edit-error-msg` — small red text for the inline error message

## Tests Added
- `src/utils/__tests__/taskFieldValidation.test.ts`: 20 tests across 4 describe blocks
  - `validateTaskName`: empty string, whitespace-only, tab-only, trimmed-valid, valid
  - `validateDuration`: valid, min-valid (1), zero, negative, non-numeric, empty string, float
  - `validateEndDateAfterStart`: valid range, equal dates, end-before-start, empty start, empty end
